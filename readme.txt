
■空のプロジェクトを新規作成する

A1. Visual Studio を起動し、[ファイル|新規作成|プロジェクト]をクリック
A2. 「新しいプロジェクト」ダイアログが出るので、左のツリーから[インストール済み/Visual C++/Windowsデスクトップ]を選ぶ
A3. 出てきた一覧から「Windowsデスクトップ ウィザード」を選択し、保存先を決めて[OK]を押す（ここでは仮に MyProject を C:\MyProject に作ったものとする）
A4. 「Windowsデスクトッププロジェクト」ダイアログが出るので、「アプリケーションの種類」を 「Windowsアプリケーション(.exe)」にして、「空のプロジェクト」だけにチェックがついている状態にする。他のチェックはすべて外す。次のような状態になっていることを確認する：

	アプリケーションの種類: 「Windowsアプリケーション(.exe)」
	空のプロジェクト: チェックあり　（チェックを外すと、余計な .cpp ファイルがついてくる）
	セキュリティ開発ライフサイクル: チェック無し (チェックを入れると、非推奨な古い文字列関数 strcpy に対する警告がエラーに格上げされ、コンパイルできなくなる）
	ATL: チェック無し

A5. OKボタンを押す


■エンジンのファイルをプロジェクトに追加する

B1. エクスプローラーを開き、上で作成したプロジェクトのフォルダ(C:\MyProject) を開く。中には MyProject.sln ファイルなどが入っているはず。
B2. そこに Engine フォルダを丸ごとコピーして入れておく
B3. Visual Studio に戻り、ソリューションエクスプローラーを表示させる
B4. ソリューションエクスプローラ内の "MyProject" 項目を右クリックし、「追加|新しいフィルター」を選び、適当な名前（ここでは仮に "mo" としておく）で決定する。
B5. 上で追加したフィルター "mo" に、さきほど C:\MyProejct にコピーした Engine の中にある core フォルダ (C:\MyProject\Engine\core) を丸ごとドラッグドロップしてプロジェクトにソースを追加する
B6. 必要ならば、 C:\MyProject\Engine\libs フォルダも同様に追加しておく


■ ソースファイルを追加する

C1. ソリューションエクスプローラ内の "MyProject" 項目を右クリックし、「追加|新しい項目」を選び、「C++ファイル(.cpp)」を選んで「追加」をクリックする。ここでは仮に "main.cpp" という名前で保存したものとする
C2. プソリューションエクスプローラ内の "MyProject" 項目を右クリックし、「プロパティ」を選び、プロパティウィンドウを開く。
C3.「構成プロパティ/リンカー/システム」を選び、サブシステムを「Windows (/SUBSYSTEM:WINDOWS)」に設定する。
  ※この作業は省略可能。省略した場合は「Console (/SUBSYSTEM:CONSOLE)」になっているので、以下のサンプルで WinMain(...) ではなく int main() と書く

C4.「構成プロパティ/C/C++/コード生成」を選び、ランタイムライブラリを「マルチスレッド デバッグ(/MTd)」を選ぶ
  ※この作業は省略可能。省略した場合は「マルチスレッド デバッグ DLL (/MDd)」になっているので、作成した exe を他のＰＣで実行するときに追加のDLLを要求される可能性があるく

C5.「構成プロパティ/C/C++/プリプロセッサ」を選び、プロセッサの定義に _CRT_SECURE_NO_WARNINGS を追加する
  ※この作業は省略可能。省略した場合はコンパイル時に 「warning C4996 ... 」という警告が大量に出るが、コンパイル自体は成功する


■ コンパイルできることを確認する

D1. main.cpp に以下の内容をそのままコピペする

#include <Windows.h>
#include "engine/core/mo.h"
int WINAPI WinMain(HINSTANCE hCurrInstance, HINSTANCE hPrevInstance, LPSTR lpCmdLine, int nCmdShow) {
	return 0;
}

D2. 「デバッグ|デバッグの開始」を選ぶか、F5を押してコンパイル＆実行し、エラーが起きないことを確認する。
エラーが起きずに実行できたらOK （何もしないプログラムなので実行しても一瞬で終わる）
※ここで、「未解決のシンボル _main が関数 ....」 というエラーが出る場合、サブシステムが「Windows」になっていることを確認する（手順 C3 を参照）
※ここで、「warning C4996: 'strcpy': This function なんたらかんたら ...」が大量に出るのが気になる場合は、プリプロセッサで _CRT_SECURE_NO_WARNINGS を定義しておく （手順 C4 を参照）
